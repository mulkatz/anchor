# Psychological Profile System

**Status:** Implemented (Testing Phase)
**Backend Only:** Not user-visible
**Framework:** CBT-focused with longitudinal tracking

---

## Overview

The Psychological Profile system creates a **Living Clinical Document** for each user - a comprehensive, professional-grade psychological case formulation that synthesizes all app data. This profile powers AI personalization while maintaining therapeutic integrity.

### Key Characteristics

- **Hybrid Approach:** Clinical depth + practical utility for AI personalization
- **Backend Only:** Used to enhance AI responses, not shown to users
- **CBT-Focused:** Aligns with Illuminate/distortion tracking
- **Append-Only History:** Weekly notes are never deleted
- **Version Tracking:** All revisions are logged with rationale

---

## Document Structure

The profile consists of 4 major sections:

### 1. Core Identity (Stable)

Changes rarely - represents fundamental understanding of the user.

```typescript
interface CoreIdentity {
  presentingConcerns: {
    primary: string; // e.g., "Generalized anxiety with social components"
    secondary: string[]; // e.g., ["Sleep disturbances", "Work stress"]
    evolutionNotes: string;
  };

  formulation: {
    cognitivePatterns: string; // Core beliefs, thinking styles
    emotionalProfile: string; // How they experience/express emotions
    physiologicalResponses: string; // Body-based anxiety manifestations
    behavioralPatterns: string; // Avoidance, safety behaviors, coping
    environmentalFactors: string; // Work, relationships, stressors
  };

  strengths: {
    personal: string[]; // e.g., ["Self-aware", "Seeks help proactively"]
    relational: string[]; // e.g., ["Strong support system"]
    coping: string[]; // e.g., ["Uses exercise", "Journaling helps"]
    insights: string[]; // e.g., ["Recognizes catastrophizing"]
  };

  riskFactors: {
    historicalFactors: string[];
    currentVulnerabilities: string[];
    protectiveFactors: string[];
    crisisHistoryCount: number;
  };
}
```

### 2. Dynamic State (Weekly Updates)

Changes frequently - represents current therapeutic landscape.

- **Current Focus:** Active themes, recent triggers, working hypotheses
- **Distortion Profile:** Patterns with trends (improving/stable/worsening)
- **Emotional Profile:** Frequent emotions, intensity trends, regulation strategies
- **Engagement Metrics:** App usage patterns, preferred features
- **Treatment Considerations:** Effective/ineffective interventions, suggested adaptations

### 3. Historical Chronicle (Append-Only)

Never deleted - maintains complete therapeutic history.

- **Weekly Notes:** Comprehensive clinical notes for each week
- **Milestones:** Breakthroughs, setbacks, insights, goal achievements
- **Assessment History:** All anxiety assessment scores over time
- **Revision Log:** When and why core understanding changed

### 4. Computed Metrics

Calculated aggregates for quick reference.

- Progress score (0-100) with trend
- Days on platform
- Total interactions
- Weekly averages
- Primary distortion patterns

## Data Sources

The profile synthesizes data from ALL app features:

| Feature                  | Data Used                                                        |
| ------------------------ | ---------------------------------------------------------------- |
| **Deep Talk (Chat)**     | Conversation themes, crisis indicators, message content          |
| **Illuminate**           | Situations, thoughts, emotions, distortions, reframes, intensity |
| **Tide Log**             | Mood depth (0-100), weather metaphors, daily notes               |

| **Depths (Journal)**     | Free-form writing, word counts                                   |
| **Assessments**          | GAD-7 scores, severity classifications                           |

## Weekly Note Structure

Each weekly clinical note contains:

```typescript
interface WeeklyNote {
  weekStartDate: string; // YYYY-MM-DD (Monday)

  dataSummary: {
    conversationCount: number;
    messageCount: number;
    illuminateEntryCount: number;
    diveSessionsCompleted: string[];
    dailyLogCount: number;
    journalSessionCount: number;
    averageEmotionalIntensity: number | null;
    crisisResourcesShown: number;
  };

  observations: {
    weekInReview: string; // 2-3 paragraph clinical summary
    notablePatterns: string[];
    progressIndicators: string[];
    concernIndicators: string[];
    quotableInsights: string[]; // Exact user quotes showing insight
  };

  changeFromLastWeek: {
    intensityChange: number; // -100 to +100
    engagementChange: 'increased' | 'stable' | 'decreased';
    newThemesEmerged: string[];
    resolvedThemes: string[];
    newDistortionsObserved: CognitiveDistortion[];
    distortionImprovements: CognitiveDistortion[];
  };

  forwardLooking: {
    suggestedFocus: string[];
    adaptationsToTry: string[];
    monitorFor: string[];
  };
}
```

## Cognitive Distortions Tracked

The profile tracks these 10 CBT cognitive distortions:

| Distortion            | Description                                         |
| --------------------- | --------------------------------------------------- |
| `catastrophizing`     | Imagining worst possible outcomes                   |
| `mind_reading`        | Assuming you know what others think                 |
| `fortune_telling`     | Predicting negative futures without evidence        |
| `all_or_nothing`      | Black-and-white thinking                            |
| `emotional_reasoning` | "I feel it, so it must be true"                     |
| `should_statements`   | Rigid rules about how things "should" be            |
| `labeling`            | Attaching negative labels to self or others         |
| `personalization`     | Taking excessive responsibility for external events |
| `filtering`           | Focusing only on negative aspects                   |
| `overgeneralization`  | Drawing broad conclusions from single events        |

## Technical Notes

- **Token Management:** Profile can grow large; views are compressed for different contexts
- **Minimum Data:** Requires at least 3 data points to create initial profile
- **Archive Strategy:** Weekly notes older than 52 weeks are moved to subcollection
- **Bilingual:** Full support for English (en-US) and German (de-DE)
- **Gemini Model:** Uses `gemini-2.5-flash` with temperature 0.4 for consistent analysis
